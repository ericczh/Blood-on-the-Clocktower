❯ 我刚刚的操作是在干什么？最后一步有什么用，还有现在我还没买域名，域名在哪里加，还有我本地的隧道要一直开着吗？为啥会有本地隧道进程                                                    
                                                                                                                                   
⏺ 从头解释一遍你在干什么                                                                                                                                                              
                                                                                                                                  
  你刚刚做了什么                                                                                                                                                                      
                                                                                                                                                                                      
  你用一个 Token(就是那串 eyJh... Base64)把你这台 Mac 注册成了 Cloudflare Tunnel 的一个"连接器"。                                                                                     
                                                                                                                                  
  整个链路是这样:                                                                                                                                                                     
                                                                                                                                  
  [公网用户的浏览器]
         ↓ https://xxx.yourdomain.com                                                                                                                                                 
  [Cloudflare 全球边缘网络]   ← Cloudflare 自己的服务器                                                                                                                               
         ↕ QUIC 长连接(加密隧道)                                                                                                                                                      
  [你 Mac 上的 cloudflared 进程] ← 就是 PID 21785 那个                                                                                                                                
         ↓ localhost:5555                                                                                                                                                             
  [你的 Flask 应用]                                                                                                                                                                   
                                                                                                                                                                                      
  这个 Token 不是随便一串字符。它是你(或者之前有人)在 Cloudflare Zero Trust 后台建 Tunnel 时生成的,包含了:                                                                            
  - Tunnel ID(那个 3965256f-... UUID)                                                                                                                                                 
  - 账号 ID                                                                                                                                                                           
  - 一个签名密钥                                                                                                                  
                                                                                                                                                                                      
  cloudflared 拿着这个 token 就知道"我是谁、连到哪个账号的哪条 tunnel"。                                                                                                              
                                                                                                                                                                                      
  ⚠️  这个 token 等同于密码,别贴到公开地方。你刚才在我们对话里贴了,严格说应该去 Cloudflare 后台把这条 tunnel 的 token 重新 rotate 一次更安全(Tunnel 页面 → Configure → Rotate          
  token)。自用工具问题不大,提醒一下。                                                                                                                                                 
                                                                                                                                                                                      
  "最后一步"(Public Hostname)是干什么的                                                                                                                                               
   
  现在隧道是搭好了但两头没接的状态:                                                                                                                                                   
                                                                                                                                  
  - 一头已经接上:cloudflared ↔ Cloudflare 边缘(你看到的 lax01、lax08 两条连接)✅                                                                                                      
  - 另一头还空着:Cloudflare 不知道"当有人访问 xxx.yourdomain.com 时,该把请求扔进这条隧道的哪个本地端口"❌                         
                                                                                                                                                                                      
  Public Hostname 就是告诉 Cloudflare:                                                                                            
                                                                                                                                                                                      
  ▎ "收到对 botc.yourdomain.com 的请求时,通过这条隧道发给对端的 localhost:5555"                                                                                                       
   
  配完这一步,整条路才通。没配的话,隧道就是个空转的连接,没任何流量走它。                                                                                                               
                                                                                                                                  
  关于域名:现在没买怎么办?                                                                                                                                                            
                                                                                                                                  
  必须先有一个接入 Cloudflare 的域名才能加 Public Hostname,这是硬门槛,绕不过去。几条路:                                                                                               
                                                                                                                                  
  路径 A:腾讯云/阿里云/Namecheap 买一个(推荐)                                                                                                                                         
  1. 任意注册商买个 .com(¥70/年)、.xyz(¥10/年首年)都行                                                                            
  2. Cloudflare Dashboard → Add a site → 输入域名 → 选 Free 套餐                                                                                                                      
  3. Cloudflare 给你两个 NS 地址                                                                                                  
  4. 回注册商控制台,把 DNS 服务器改成这两个 NS                                                                                                                                        
  5. 等生效(5 分钟 ~ 几小时)                                                                                                                                                          
  6. 然后才能去 Public Hostname 里选到这个域名                                                                                                                                        
                                                                                                                                                                                      
  路径 B:Cloudflare Registrar 直接买(最省事)                                                                                                                                          
  Cloudflare 自家也卖域名,成本价(.com 差不多 $10/年、没有"首年便宜、续费涨价"那套把戏),买完自动就在 Cloudflare 了,跳过改 NS 那步。Dashboard 左侧 Domain Registration                  
  里。唯一缺点:不支持中国大陆支付方式,得有 Visa/Master 国际卡。                                                                                                                       
                                                                                                                                                                                      
  路径 C:先用 Cloudflare 免费子域占坑                                                                                                                                                 
  Cloudflare Zero Trust 里建 Application 时,没有自己域名的话可以临时用 *.cfargotunnel.com 的内部地址,但那个只在 Access 认证场景下好用,公开 HTTP 不方便。不推荐,还不如花 10 块买个
  .xyz。                                                                                                                                                                              
                                                                                                                                  
  建议:买个 .xyz 或 .top 这种便宜的(一年几块钱),先跑通,以后要正式用再换。                                                                                                             
                                                                                                                                  
  本地隧道进程要一直开着吗?                                                                                                                                                           
                                                                                                                                  
  对,必须一直开着。这就是它的工作原理 —— cloudflared 主动从你 Mac 连出去 到 Cloudflare,保持长连接。Cloudflare 收到公网请求后,把请求沿着这条已有的长连接反向送回到你 Mac。             
   
  - cloudflared 关了 → 长连接断 → Cloudflare 那边 Tunnel 状态变 DOWN → 访问 xxx.yourdomain.com 返回 Error 1033(tunnel offline)                                                        
  - 你 Mac 关机 / 休眠 → 同上                                                                                                     
  - 网络断 → 自动重连,恢复后继续用                                                                                                                                                    
                                                                                                                                                                                      
  所以它为什么叫"Tunnel":因为这根管子必须一直插着两头才能通水。                                                                                                                       
                                                                                                                                                                                      
  为啥会有"本地隧道进程"                                                                                                                                                              
                                                                                                                                  
  因为你家里这台 Mac 没有公网 IP(你前面看到的 198.18.0.1 是 Clash 的 fake-ip,不是真公网)。公网上的浏览器没法直接找到你这台机器。                                                      
   
  传统做法是这样的:                                                                                                                                                                   
  - 买台公网服务器 → 装 nginx → 做反向代理 → 但你家里 Mac 还是得有办法让那台服务器访问到(内网穿透/frp/wireguard...)               
                                                                                                                                                                                      
  Cloudflare Tunnel 把这事做成了"出站打洞":
  - 你 Mac 主动出去连 Cloudflare(出站流量一般不被路由器/防火墙拦)                                                                                                                     
  - 公网请求从 Cloudflare 那边"塞进"这条已有连接                                                                                                                                      
  - 不用公网 IP、不用开路由器端口、不用 DDNS、HTTPS 自动有                                                                                                                            
                                                                                                                                                                                      
  本质就是花一个本地常驻进程,换一个公网入口。这个进程不能停 —— 停了就没入口了。                                                                                                       
                                                                                                                                                                                      
  你现在要做的事(按顺序)                                                                                                                                                              
                                                                                                                                                                                      
  1. 买一个域名(哪家都行,能改 NS 就行)                                                                                                                                                
  2. 在 Cloudflare 添加站点,把域名 NS 指过来,等生效                                                                               
  3. Zero Trust → Networks → Tunnels,找到你那条 tunnel,Public Hostname → Add,填:                                                                                                      
    - Subdomain: 随你喜欢(botc)                                                                                                                                                       
    - Domain: 下拉选你刚接入的域名                                                                                                                                                    
    - Service: HTTP                                                                                                                                                                   
    - URL: localhost:5555                                                                                                                                                             
  4. 起 Flask,访问 https://botc.<你域名>/ 看到登录页 = 成功                                                                       
                                                                                                                                                                                      
  本地这个 cloudflared 服务装过一次就不用管了,开机自动启,后台常驻。你只需要管 Flask 进程。

❯ 如果我换了电脑怎么办                                                                                                                                                                
                                                                                                                                    
⏺ 换电脑很简单 —— 隧道本身在云端,不绑电脑                                                                                                                                             
                                                                                                                                    
  核心概念:你看到的 NYC_botc_prod 这条 Tunnel、Tunnel ID、Hostname routes 配置,全在 Cloudflare 云端。本地 cloudflared 只是一个"连接器"(Connector),可以在任意数量的机器上跑,作用都是把 
  Cloudflare 的流量转发到本机的某个端口。                                                                                                                                             
                                                                                                                                                                                      
  所以换电脑 = 换一个新 Connector,Tunnel 和域名配置一行都不用改。                                                                                                                     
   
  场景 A:旧电脑还在,新电脑接管(推荐做法)                                                                                                                                              
                                                                                                                                    
  新电脑上:                                                                                                                                                                           
                                                                                                                                    
  brew install cloudflared                                                                                                                                                            
  sudo cloudflared service install <你的 token>   # 同一个 token 复用                                                               
                                                                                                                                                                                      
  就这两条命令。新电脑的 cloudflared 启动后会自动加入同一条 tunnel,你的 Connectors 列表就会变成 2 个(看截图那个列表会多一行)。                                                        
                                                                                                                                                                                      
  Cloudflare 会自动在两个 Connector 之间做负载均衡和故障转移:                                                                                                                         
                                                                                                                                    
  - 两台都开机 → 流量随机分配,任意一台挂了另一台顶上(高可用)                                                                                                                          
  - 只开新电脑 → 流量全走新电脑                                                                                                     
  - 只开旧电脑 → 流量全走旧电脑                                                                                                                                                       
                                                                                                                                    
  Flask 应用要在哪台就把哪台的 cloudflared 跑起来就行。因为 Cloudflare 只负责送到 localhost:5555,这个 localhost 是各自机器的 localhost。                                              
                                                                                                                                    
  彻底迁移完成后,旧电脑上卸载:                                                                                                                                                        
                                                                                                                                    
  sudo cloudflared service uninstall
                                                                                                                                                                                      
  Cloudflare 后台 Connectors 列表里旧的那条状态会变 Disconnected,点旁边删掉即可。                                                                                                     
                                                                                                                                                                                      
  场景 B:旧电脑丢了/坏了/没了                                                                                                                                                         
                                                                                                                                    
  一样,新电脑上:                                                                                                                                                                      
                                                                                                                                    
  brew install cloudflared
  sudo cloudflared service install <你的 token>
                                                                                                                                                                                      
  完事。Cloudflare 看到新 Connector 上来,自动接管流量。公网 URL https://botc.<你域名>/ 一秒不变。                                                                                     
                                                                                                                                                                                      
  Token 在哪里找                                                                                                                                                                      
                                                                                                                                    
  如果你忘了当初那串 eyJh...:                                                                                                                                                         
                                                                                                                                    
  Cloudflare Zero Trust → Networks → Tunnels → NYC_botc_prod → Configure → 有个 "Install connector" 区块,里面显示 token 和一键安装命令(复制粘贴即可)。                                
                                                                                                                                    
  或者在顶部 ... (更多菜单) → Refresh token 重新生成一个新 token(旧的立即失效,老机器上的连接会断,适合旧电脑丢了安全起见)。                                                            
                                                                                                                                    
  会丢什么?不会丢什么?
